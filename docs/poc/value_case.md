# WinbackFlow PoC Value Case

## Problem

Most D2C teams already have email/SMS tooling but still struggle with:

- slow manual segmentation
- generic winback campaigns
- weak explainability of targeting choices
- unclear lift measurement versus baseline.

## Solution

WinbackFlow adds an orchestration layer that:

1. identifies low-engagement customers
2. suggests next-best-action with confidence and rationale
3. requires human approval before outreach
4. captures telemetry for outcome measurement.

## Why this is different

- Approval-first workflow, not blind automation.
- KPI design includes action rate and projected recovered revenue.
- Lightweight integration path: CSV first, connectors later.
- Deterministic demo and repeatable pilot runbook.

## Investor demo outcomes

- Demonstrates workflow viability in one command.
- Shows measurable output artifacts (`analytics/events.jsonl`, `analytics/poc_summary.json`).
- Provides path to production via adapter-based integrations.
